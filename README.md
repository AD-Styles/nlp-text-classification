## 📌 프로젝트 요약 (Project Overview)
Sentence-Transformers와 머신러닝(Logistic Regression)을 활용하여 자연어의 의미를 파악하고 자동으로 분류하는 NLP 프로젝트. 
1. **[이진 분류]** 영화 리뷰 데이터를 활용하여 텍스트의 긍정/부정 감성을 판별하는 기초 파이프라인을 설계.
2. **[다중 분류]** 동일한 임베딩 구조를 확장하여, 국민청원 데이터를 17개의 복잡한 사회적 카테고리로 세분화하는 심화 모델을 구축하고 데이터 불균형 문제를 분석.


## 🎯 핵심 목표 (Motivation)
| 구분 | 세부 내용 |
| :--- | :--- |
| **분류 모델의 확장** | 단순한 이진 분류(감성 분석)에서 시작하여 복잡한 다중 분류(카테고리 예측)로 기술을 점진적으로 확장. |
| **자연어 처리(NLP) 기초 구현** | 텍스트 데이터를 컴퓨터가 연산할 수 있는 다차원 벡터로 변환하는 임베딩 과정을 코드로 구현. |
| **데이터 불균형 분석** | 17개 카테고리로 구성된 현실 데이터(국민청원)의 불균형이 모델 성능에 미치는 영향을 확인. |


## 🚨 개념 및 파이프라인 (Concept & Pipeline)
| 항목 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; | 세부 내용 |
| :--- | :--- |
| **핵심 개념**<br>**(임베딩)** | 자연어 텍스트의 의미를 다차원 공간의 좌표(벡터)로 변환하는 기술입니다. 문장의 의미가 비슷할수록 벡터 공간에서 가까운 위치에 매핑. |
| **사용**<br>**알고리즘** | `paraphrase-multilingual-MiniLM-L12-v2` 모델 (텍스트 임베딩), `LogisticRegression` (분류기) |
| **학습**<br>**파이프라인** | 1. 텍스트 데이터 정제 및 라벨 인코딩<br>2. 임베딩 모델을 통한 텍스트 벡터화 (`model.encode`)<br>3. 로지스틱 회귀 모델 학습 (`clf.fit`)<br>4. 성능 검증 및 새로운 문장 테스트 |

## 2. 실험 결과 및 분석 (Results & Analysis)
| 환경 및 분류 체계 | 테스트<br> 정확도 | 엔지니어링 분석 포인트 |
| :--- | :--- | :--- |
| **영화 리뷰<br> 감성 분석**<br>(이진 분류) | **약 78.5%** | 긍정과 부정, 두 가지 명확한 클래스로 나뉘어 있어 모델이 특징을 쉽게 학습하고 안정적인 성능을 도출. |

## 3. 회고록 (Retrospective)
### 💡 Keep (좋았던 점 & 유지할 점)
- **일관된 파이프라인 설계**: 데이터 로드부터 임베딩, 분류, 예측 함수까지 두 가지 다른 과제에 동일한 아키텍처 흐름을 적용하여 코드의 재사용성과 파이프라인에 대한 이해도를 높였습니다.
- **다양한 환경 테스트**: 쉬운 난이도(이진 분류)에서 시작해 현실적이고 복잡한 난이도(다중 분류)로 점진적 테스트 수행하며 머신러닝 모델의 한계점을 명확히 관찰할 수 있었습니다.

### 🧗 Problem (아쉬웠던 점 & 직면한 문제)
- **데이터 불균형 문제**: 국민청원 데이터의 경우 카테고리별 데이터 수가 균일하지 않아 정확도가 예상보다 낮게 측정되었습니다.
- **표면적 단어의 한계**: 문장의 깊은 뉘앙스나 반어법을 처리하기에는 단순 문장 임베딩과 로지스틱 회귀 조합에 한계가 있음을 체감했습니다.

### 🚀 Try (다음 프로젝트에서 시도할 점)
- **데이터 증강 및 샘플링 기법 도입**: 다중 분류 모델의 정확도 향상을 위해 SMOTE와 같은 오버샘플링 기법을 적용하거나 데이터 가중치를 조정해 볼 계획입니다.
- **고도화된 모델 아키텍처 학습**: 문맥의 흐름을 더 깊이 이해할 수 있도록 BERT, RoBERTa와 같은 트랜스포머 기반의 미세조정(Fine-Tuning) 모델을 다음 포트폴리오의 주제로 삼아 성능을 비교해 보고자 합니다.
