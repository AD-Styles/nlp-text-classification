## 📌 프로젝트 요약 (Project Overview)
Sentence-Transformers와 머신러닝(Logistic Regression)을 활용하여 자연어의 의미를 파악하고 자동으로 분류하는 NLP 프로젝트
1. **[영화 리뷰 감성 분석기]** - 이진 분류: 영화 리뷰 데이터를 활용하여 텍스트의 긍정/부정 감성을 판별하는 기초 파이프라인을 설계
2. **[국민청원 카테고리 분류기]** - 다중 분류: 동일한 임베딩 구조를 확장하여, 국민청원 데이터를 17개의 복잡한 사회적 카테고리로 세분화하는 심화 모델을 구축하고 데이터 불균형 문제를 분석

## 🎯 핵심 목표 (Motivation)
| 구분 | 세부 내용 |
| :--- | :--- |
| **분류 모델의 확장** | 단순한 이진 분류(감성 분석)에서 시작하여 복잡한 다중 분류(카테고리 예측)로 기술을 점진적으로 확장 |
| **자연어 처리(NLP) 구현** | 텍스트 데이터를 컴퓨터가 연산할 수 있는 다차원 벡터로 변환하는 임베딩 과정을 코드로 구현 |
| **데이터 불균형 분석** | 17개 카테고리로 구성된 현실 데이터(국민청원)의 불균형이 모델 성능에 미치는 영향을 확인 |

## 🚨 개념 및 파이프라인 (Concept & Pipeline)
| 항목 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; | 세부 내용 |
| :--- | :--- |
| **핵심 개념**<br>**(임베딩)** | 자연어 텍스트의 의미를 다차원 공간의 좌표(벡터)로 변환하는 기술입니다. 문장의 의미가 비슷할수록 벡터 공간에서 가까운 위치에 매핑 |
| **사용**<br>**알고리즘** | `paraphrase-multilingual-MiniLM-L12-v2` 모델 (텍스트 임베딩), `LogisticRegression` (분류기) |
| **학습**<br>**파이프라인** | 1. 텍스트 데이터 정제 및 라벨 인코딩<br>2. 임베딩 모델을 통한 텍스트 벡터화 (`model.encode`)<br>3. 로지스틱 회귀 모델 학습 (`clf.fit`)<br>4. 성능 검증 및 새로운 문장 테스트 |
| **확률 예측**<br>**(Top-K)** | 단순 정답 도출을 넘어 `predict_proba`를 활용해 각 클래스에 속할 확률(Confidence Score)을 계산하고 최상위 K개의 결과를 반환 |

## 2. 실험 결과 및 분석 (Results & Analysis)
| 환경 및 분류 체계 | 테스트<br> 정확도 | 엔지니어링 분석 포인트 |
| :--- | :--- | :--- |
| **영화 리뷰<br> 감성 분석**<br>(이진 분류) | **약 78.5%** | 긍정과 부정, 두 가지 명확한 클래스로 나뉘어 있어 모델이 특징을 쉽게 학습하고 안정적인 성능을 도출 |

## 💡 회고록 (Retrospective)
&emsp;이번 실습을 통해서 텍스트를 컴퓨터가 이해할 수 있는 다차원 벡터로 바꾸는 '임베딩' 과정의 흐름을 파악할 수 있었습니다. 영화 리뷰와 국민청원이라는 완전히 다른 두 가지 데이터셋에 동일한 파이프라인(전처리 ➔ 임베딩 ➔ 로지스틱 회귀)을 적용해 보면서, 자연어 처리의 전체적인 흐름을 파악할 수 있었습니다. 특히 단순한 이진 분류를 넘어 17개나 되는 다중 분류로 모델을 확장해 나가는 과정에서 큰 성취감을 느꼈습니다.
<br>&emsp;하지만 이번 실습으로 현실 데이터의 장벽과 문맥 이해의 한계를 깨달았습니다. 현실의 데이터는 생각보다 호락호락하지 않았습니다. 국민청원처럼 특정 분야에만 글이 몰려 있는 '데이터 불균형' 상황에서는 모델이 소수 카테고리의 특징을 제대로 학습하지 못해 정확도가 떨어지는 아쉬움이 있었습니다. 또한, 반어법이 섞인 문장의 경우 단순히 표면적인 단어만으로는 벡터 공간 상에서 올바른 위치(의미)를 찾지 못한다는 점을 확인하며, 단순한 문장 임베딩과 선형 분류 모델이 가진 문맥 파악의 명확한 한계를 느꼈습니다.
<br>&emsp;더 나아가, 단순히 문장 전체를 막연하게 임베딩하는 방식에서 벗어나 문장 속 단어들의 전후 맥락을 깊이 있게 이해할 수 있는 최신 트랜스포머 모델을 파인튜닝(Fine-Tuning)해보고 싶다는 생각을 하게되었습니다. 이를 통해 반어법의 미묘한 뉘앙스까지 캐치할 수 있는 훨씬 똑똑한 분류기를 만들어 볼 계획입니다.
