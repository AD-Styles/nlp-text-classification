## 📌 프로젝트 요약 (Project Overview)
Sentence-Transformers와 머신러닝(Logistic Regression)을 활용하여 자연어의 의미를 파악하고 자동으로 분류하는 NLP 프로젝트
1. **[영화 리뷰 감성 분석기]** - 이진 분류: 영화 리뷰 데이터를 활용하여 텍스트의 긍정/부정 감성을 판별하는 기초 파이프라인을 설계
2. **[국민청원 카테고리 분류기]** - 다중 분류: 동일한 임베딩 구조를 확장하여, 국민청원 데이터를 17개의 복잡한 사회적 카테고리로 세분화하는 심화 모델을 구축하고 데이터 불균형 문제를 분석

## 🚨 개념 및 파이프라인 (Concept & Pipeline)
| 항목 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; | 세부 내용 |
| :--- | :--- |
| **핵심 개념**<br>**(임베딩)** | 자연어 텍스트의 의미를 다차원 공간의 좌표(벡터)로 변환하는 기술입니다. 문장의 의미가 비슷할수록 벡터 공간에서 가까운 위치에 매핑 |
| **사용**<br>**알고리즘** | `paraphrase-multilingual-MiniLM-L12-v2` 모델 (텍스트 임베딩), `LogisticRegression` (분류기) |
| **학습**<br>**파이프라인** | 1. 텍스트 데이터 정제 및 라벨 인코딩<br>2. 임베딩 모델을 통한 텍스트 벡터화 (`model.encode`)<br>3. 로지스틱 회귀 모델 학습 (`clf.fit`)<br>4. 성능 검증 및 새로운 문장 테스트 |
| **확률 예측**<br>**(Top-K)** | 단순 정답 도출을 넘어 `predict_proba`를 활용해 각 클래스에 속할 확률(Confidence Score)을 계산하고 최상위 K개의 결과를 반환 |

## 💡 회고록 (Retrospective)
&emsp;이번 실습을 통해서 텍스트를 컴퓨터가 이해할 수 있는 다차원 벡터로 바꾸는 '임베딩' 과정의 흐름을 파악할 수 있었습니다. 영화 리뷰와 국민청원이라는 완전히 다른 두 가지 데이터셋에 동일한 파이프라인(전처리 ➔ 임베딩 ➔ 로지스틱 회귀)을 적용해 보면서, 자연어 처리의 전체적인 흐름을 파악할 수 있었습니다. 특히 단순한 이진 분류를 넘어 17개나 되는 다중 분류로 모델을 확장해 나가는 과정에서 큰 성취감을 느꼈습니다.
<br>&emsp;하지만 이번 실습으로 현실 데이터의 장벽과 문맥 이해의 한계를 깨달았습니다. 현실의 데이터는 생각보다 호락호락하지 않았습니다. 영화 리뷰 감성 분석기는 텍스트 분류의 가장 기초적인 이진 분류(Binary Classification) 프로젝트였지만, 실제 데이터를 다루며 '언어의 모호성'과 '수치화의 한계'를 깊이 있게 체감할 수 있는 과정이었습니다. 반어법이 섞인 문장의 경우 단순히 표면적인 단어만으로는 벡터 공간 상에서 올바른 위치(의미)를 찾지 못한다는 점을 확인하였으며 또한, 긍정/부정의 표현이 명확하지 않거나 중립적인 내용을 담고 있는 문장을 AI 가 헷갈려 하는것을 확인하였습니다. 영화 리뷰 분석기는 NLP의 'Hello World'와 같지만, 그 속에서 데이터 불균형과 문맥 파악의 난제라는 현실적인 벽을 마주할 수 있었습니다. 특히 긍정/부정의 경계가 모호한 중립적인 문장을 어떻게 처리할지에 대한 고민은, 향후 더 정교한 감성 분석 엔진을 설계하는 데 있어 소중한 자산이 되었습니다.
<br>&emsp;국민청원 데이터셋은 단순한 긍정/부정을 넘어 17개의 복잡한 사회적 맥락을 분류해야 하기에, 기술적으로 고민할 지점이 훨씬 많았습니다. 국민청원처럼 특정 분야에만 글이 몰려 있는 '데이터 불균형' 상황에서는 모델이 소수 카테고리의 특징을 제대로 학습하지 못해 정확도가 떨어지는 아쉬움이 있었습니다. '인권/성평등' 카테고리는 데이터가 넘쳐나는 반면, 상대적으로 적은 카테고리는 모델이 충분히 학습할 기회를 얻지 못했습니다. 그 결과 모델은 모르는 문장이 나오면 일단 데이터가 많은 쪽으로 예측하려는 '편향(Bias)'을 보였습니다. 78.5%의 이진 분류 성능에 비해 44.2%라는 수치는 실망스러울 수 있지만, 이는 알고리즘의 한계라기보다 데이터 분포의 문제라는 점을 배웠습니다. 이를 해결하기 위해 추후 데이터 증강(Augmentation)이나 클래스 가중치(Class Weight) 조절이 필수적임을 깨달았습니다. 더 나아가, 단순히 선형 분류기를 쓰는 것을 넘어, 문장의 전후 맥락을 더 깊게 파악할 수 있는 Self-Attention 기반의 트랜스포머 모델을 직접 파인튜닝해야 하는 강력한 동기가 되었습니다.
